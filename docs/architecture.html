paper_trail-human

Arquitetura Hexagonal (Ports & Adapters)

Visão Geral

Transforma PaperTrail::Version em hashes legíveis para UIs de auditoria. O domínio de formatação é isolado de dependências externas através de ports e adapters.

Public API
PaperTrail::Human.format(version)
Formata um Version em hash estruturado
PaperTrail::Human.format_collection(versions)
Formata uma coleção de Versions
PaperTrail::Human.configure { |c| }
DSL de configuração por model
▼
Core (Domain) — Zero dependências externas
Presenter
Orquestra formatação completa. Recebe version + config, retorna hash. Stateless.
FieldFormatter
Formata campo individual. Resolve nome humano + aplica resolver configurado.
ChangeExtractor
Extrai changes de Version. Suporta JSON, YAML e Hash (jsonb).
▼
Ports (Interfaces)
Ports::Resolver
Contrato: #resolve(value) → String. Todo adapter implementa esta interface.
▼
Adapters (Implementações)
Relation
FK → nome via ActiveRecord find_by
Enum
Valor → label via classe ou mapping hash
Boolean
true/false → labels customizados
Custom
Qualquer valor → resultado de lambda

Fluxo de Dados

1
ChangeExtractor recebe PaperTrail::Version e extrai hash de mudanças
2
Presenter filtra campos ignorados e resolve whodunnit
3
FieldFormatter para cada campo: busca config → instancia resolver → resolve valores
4
Retorna hash estruturado com user, event, model, item_id, created_at, fields
{
  user: "João Silva",
  event: "update",
  model: "User",
  item_id: 1,
  created_at: 2026-05-29 12:00:00,
  fields: [
    { field: "Nome", previous_value: "João", value: "João Silva" },
    { field: "Empresa", previous_value: "Acme", value: "Globex" }
  ]
}

Decisões de Design

ADR-001: Hexagonal Architecture

Core sem dependências externas. Adapters implementam ports. Permite testar core sem banco e adicionar resolvers sem modificar core (Open/Closed).

ADR-002: Thread-Safety via Frozen Configs

Mutex protege escritas. ModelConfig frozen após registro. Leituras são lock-free. Presenter é stateless.

ADR-003: String-Based Resolver Lookup

RESOLVER_MAP com strings + Object.const_get. Core não importa adapters. Erro claro para types inválidos.

ADR-004: Graceful Fallback

Resolvers capturam NameError e retornam valor raw. Logs de auditoria nunca explodem por model deletado.

Como Estender

Novo Resolver

  • Criar lib/paper_trail/human/adapters/resolvers/novo_tipo.rb
  • Implementar include Ports::Resolver + #resolve(value)
  • Adicionar ao RESOLVER_MAP em core/field_formatter.rb
  • Adicionar require_relative em lib/paper_trail/human.rb
  • Criar spec em spec/unit/adapters/resolvers/

Customizar Nomes de Campos

PaperTrail::Human.configure do |config|
  config.field_name_resolver = ->(field_name, item_type) {
    item_type.constantize.human_attribute_name(field_name)
  }
end
